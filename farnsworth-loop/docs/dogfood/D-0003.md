# D-0003 — `fl-parse.mjs` mis-reads prose `1 grand loop` and `2x opus` shorthand

- id: D-0003
- status: open
- severity: sev3            # cosmetic/UX — surfaces as a clear STOP-and-ask error, never a wrong run
- area: parse
- discovered: 2026-06-15
- reporter: rob
- provenance (NON-DURABLE): interactive `bin/fl-parse.mjs` invocations, 2026-06-15 (no run dir).

## Problem

Two natural prose forms a user typed were not understood by the parser:

1. **`1 grand loop`** — intended as the Z (grand-loop) count, but the prose-model-spec parser tried
   to read `grand` as a model token and errored. The user must instead use the sigil `:Z` form
   (e.g. `@@FL:8:2:2`). (Aside: there is also no way to express "exactly one *implementing* loop" —
   `Z=1` is the no-implement isolated tournament and the minimum grand loop is `Z=2` = two loops —
   but that is a separate design item, not this parse bug.)
2. **`2x opus`** — the `Nx model` multiplier shorthand is not recognised; the parser needs
   `2 opus`. The `2x` form fell through to the interactive gate (`needsGate:true`) with the spec
   silently dropped.

Neither produces a *wrong* tournament (case 1 is a loud error; case 2 falls back to the gate), so
this is sev3 — but both cost a re-state and surprised the user.

## Durable evidence (verbatim excerpt — survives `.runs/` deletion)

> `fl-parse.mjs "@@FL ... 2 minimax, 1 grand loop, 5) create a persistent dogfood file ..."` →
> `"errors": [ "Unrecognised model token(s) in spec: \"grand\". Known: opus, sonnet, haiku,
>  glm[...], codex[...], minimax-m3, or a live local id. ..." ]`
>
> `fl-parse.mjs "@@FL 2x opus, 2x sonnet, 2x codex, 2x minimax, two pass ..."` →
> `{ "task": "", "n": null, "mode": 1, "assignment": null, "needsGate": true }`  (spec dropped)

## Repro

1. `node bin/fl-parse.mjs "@@FL 2 opus, 1 grand loop — do X"` → error on `grand`.
2. `node bin/fl-parse.mjs "@@FL 2x opus — do X"` → `needsGate:true`, assignment null.

## Suspected fix (for the worker who claims this)

In `bin/fl-parse.mjs`: (a) recognise a `<n> grand loop[s]` phrase as a Z directive (set `z`, strip
it from the task/spec) so prose and sigil agree; (b) accept `Nx <model>` as equivalent to
`N <model>` in the model spec. Add unit cases to the parser's test suite for both. Consider also
the deeper "no single implementing loop" design question as a separate item if it matters.
